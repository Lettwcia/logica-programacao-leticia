# logica-programacao-leticia
Conceitos essenciais (com exemplos simples)
O que é Git e por que usamos (controle de versão)

Git é uma ferramenta que registra o histórico das mudanças nos seus arquivos.
Você pode voltar no tempo, ver o que mudou e não perde versões antigas.
Exemplo: é como salvar várias versões de um trabalho: “trabalho_v1”, “v2”, “final_de_verdade”.

O que é GitHub

O GitHub é um site onde você guarda seus projetos feitos com Git e pode compartilhar com outras pessoas.
Ele funciona como uma “nuvem” para seus repositórios.
Exemplo: tipo um Google Drive, só que para projetos versionados com Git.

O que é um repositório

É a “pasta do projeto” que o Git controla.
Dentro do repositório ficam seus arquivos e o histórico de mudanças.
Exemplo: é como uma pasta no seu computador, só que com memória de tudo que já mudou ali.

O que são: commit, push, pull, branch

(definições curtas + 1 analogia)

Commit → salvar uma versão do seu trabalho no Git
Analogia: um “check-in” do que você fez até agora.

Push → enviar seus commits para o GitHub
Analogia: subir seus arquivos do PC para a nuvem.

Pull → trazer atualizações do GitHub para seu computador
Analogia: baixar a versão mais nova da nuvem.

Branch → uma linha paralela de trabalho (sem mexer na principal)
Analogia: fazer um rascunho separado antes de misturar com o trabalho final.
